## Approval Levels

### Low Risk — Auto-approved
- Documentation, reports, analysis, research, planning

### Medium Risk — Requires approval
- New features, refactors, API changes, database schema changes, dependency upgrades

### High Risk — Requires explicit approval + review
- File or data deletion, production deployments, large migrations, major rewrites, security changes

---

## How to Approve

1. Review the item in `/approvals/{ID}.md`
2. Reply: **Approved**, **Rejected**, or **Needs revision**
3. Cap DevRoom proceeds based on response
