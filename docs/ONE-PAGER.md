# Cap · DevRoom — One Pager

**Cap family · Virtual engineering office · v3.1.0**

---

## What it is

**Cap · DevRoom** is a local engineering command center where 13 codename agents (APEX, FORGE, VAULT, …) run in **sandbox copies** of your Cap portfolio — with CEO approval, deliverables awareness, and real activity telemetry.

---

## Core capabilities

| Area | What you get |
|------|----------------|
| **Command center** | Portfolio metrics, live agent roster, activity log |
| **CEO (APEX)** | Morning briefing (rules + AI), CEO command routing |
| **Agent workspaces** | Per-agent sandbox runs via Cursor SDK |
| **Governance** | Approval queue for Medium/High risk work |
| **Operations** | Tasks, priorities, knowledge base |
| **Memory** | Token-smart bullets per agent after completed jobs |
| **Safety** | Sandboxes only — never touches production paths |

---

## Why founders choose it

- **Control:** Approval pipeline before risky agent work  
- **Structure:** Engineering org chart — not one generic chat  
- **Safety:** Sandboxed portfolio copies  
- **Honesty:** Smart Assistant + Cursor SDK — no fake AI hype  
- **Access:** iPhone, iPad, Mac, PWA install  

---

## Tech

Turborepo monorepo · Next.js dashboard · Prisma SQLite · Cursor SDK · PWA

---

## Get started

1. `cd ~/Desktop/Cap-DevRoom && npm run dev:stack`  
2. Add `CURSOR_API_KEY` to `dashboard/.env.local`  
3. Open **Command center** → run agents from **Agents** → review **Approvals**  
4. On iPhone/iPad: **Share → Add to Home Screen** (see DEVICE-GUIDE.md)
